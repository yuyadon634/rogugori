# Gemini プロンプト改善スキル

## このスキルを使うタイミング

- Gemini の分析で特定のメトリクス（心拍・ペース・BMI など）への言及が薄いと感じたとき
- プロンプトを変更して分析品質を向上させたいとき
- `config.json` の `focus_metrics` や `gemini_client.py` のプロンプト構築ロジックを見直したいとき

---

## Step 1: 現状を把握する

以下のファイルを読んで現状を確認する。

1. `config.json` — `role` / `tone` / `focus_metrics` を確認
2. `src/gemini_client.py` — `_build_system_prompt` / `_format_today_data` / `_format_history` を確認

---

## Step 2: 改善原則

以下のルールに照らして、現在のプロンプトに不足がないか確認する。

### 心拍（必須言及 4 点）

| 言及すべき内容 | 判断基準 |
|---|---|
| 平均心拍 vs Zone2 基準（145 bpm）との乖離 | 乖離が 10 bpm 以上なら過負荷・未活用を指摘 |
| Zone 別比率の評価（Z1 / Z2 / Z3 分布） | Z2 比率が低ければ有酸素基盤強化を推奨 |
| 疲労兆候（心拍高値日は翌日休養推奨） | 平均心拍が直近 7 日平均 +10 bpm 以上で警告 |
| 有酸素能力トレンド（同ペース・心拍低下傾向） | 過去 7 日の ペース/心拍 比を比較して言及 |

### ペース（必須言及 3 点）

| 言及すべき内容 | 例 |
|---|---|
| 今日のペース vs サブ 4 目標（5:41/km）の差 | 「目標まであと +0:15/km」など具体的に |
| 過去 7 日の平均ペース推移（改善 / 悪化） | 先週比 +5 秒 / km など数値で |
| ペースと心拍のバランス（心拍を抑えて同ペースか） | 「同ペースで心拍が 3 bpm 下がった」など |

### BMI（必須言及 1 点）

| 言及すべき内容 | 補足 |
|---|---|
| BMI とランニングパフォーマンスの相関 | 体重 1 kg 減 ≒ フルマラソン 2〜3 分短縮の目安で具体的に言及 |

### 睡眠（言及推奨）

- 睡眠スコアと翌日推奨トレーニング強度（高スコア→高強度 OK、低スコア→軽め）

---

## Step 3: 変更箇所の判断

| 変更したい内容 | 変更するファイル |
|---|---|
| コーチのキャラクター・口調 | `config.json` の `role` / `tone` |
| 評価軸の追加・削除・表現 | `config.json` の `focus_metrics` |
| 必須言及ルールの追加・フォーマット構成 | `src/gemini_client.py` の `_build_system_prompt` |
| データの整形方法（渡すデータ項目の追加） | `src/gemini_client.py` の `_format_today_data` / `_format_history` |

## Step 5: 変更後の確認

変更後、以下を口頭または実行で確認する。

- `config.json` が valid な JSON か（構文エラーなし）
- `src/gemini_client.py` の linter エラーなし
- プロンプト全文をログで確認したい場合は `LOG_LEVEL=DEBUG` で実行すると `logger.debug` で出力される

---

## 注意事項

- `focus_metrics` の各項目は `_build_system_prompt` 内で箇条書きに変換されるため、1項目が長くなりすぎても問題ない
- LINE メッセージの文字数上限（400〜600 文字）は `_build_system_prompt` に明記済み。必須言及を追加した分、Gemini が各項目を簡潔にまとめるよう促す表現（「簡潔に」「1文で」など）を足すと良い
- Zone2 の基準 145 bpm はユーザーの年齢・最大心拍数によって変わる。変更したい場合は `config.json` に `zone2_threshold_bpm` を追加して `_build_system_prompt` で参照する設計にするとよい
